# Weekly review playbook

## Purpose

Close the week, capture loose ends, set focus for next week—without living in the calendar app.

## Preconditions

- [ ] Calendar for the week reviewed
- [ ] Inbox / capture tool at zero or triaged

## Steps

1. Open [`templates/review-weekly.md`](../../templates/review-weekly.md) in `wiki/reviews/` for this week’s file.
2. List wins and misses (5 min).
3. Pick **1–3 project focuses** for next week; everything else is backlog.
4. Skim **areas** ([[work-and-career]])—any minimums missed?
5. Optional: one sentence to [journal/2026/README](../journal/2026/README.md) if you keep a log.

## Prompts (optional)

```text
Summarize my week from these wiki updates: [paste links to journal/ project logs]
What should I stop pretending I will do next week?
```

## Outputs

- Filled weekly review note
- Updated project “Current focus” sections

## Failure modes

- **Too long:** cap at 25 minutes; use a timer.
- **No projects:** focus on areas and questions instead.
